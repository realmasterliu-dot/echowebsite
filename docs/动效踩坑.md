# 动效踩坑日志（mp-weixin）

记录时间：2026-07-21  
涉及：`CompanyIntro`（ScrollReveal / ScrollRevealText）与 `HeroSection` slogan 入场。

## 现象

| 场景 | 表现 |
|------|------|
| CompanyIntro | 内容空白占位，滚动后也不出现 |
| Hero slogan | 有文案结构，但看不到上滑/淡入过渡 |

## 根因（同一类问题）

1. **过渡驱动方式错误**：依赖 scoped CSS class 切换 `opacity`/`transform`，微信小程序常跳过 transition，等于瞬切或未应用。
2. **时序错误**：隐藏态尚未完成绘制就设为显示 → 浏览器/小程序跳过过渡；若触发逻辑失败则永远停在 `opacity: 0` → 空白。
3. **单位/变量不可靠**：`translateY(%)`、CSS 自定义属性参与 `transition`/`transform` 在 MP 上易失效。
4. **滚动监听未挂到页面**：仅在子组件 `import { onPageScroll }` 时，页面未声明则回调不触发（uni-app 性能优化限制）。
5. **观察器作用域**：观察自定义组件根节点、或错误的 `createIntersectionObserver` 上下文，导致永不回调。
6. **附带**：删除/重命名组件后 DevTools 仍请求旧路径 → `ENOENT`（与过渡无关，但会掩盖真问题）。处理：在 `pages/home/index.vue` 用 `v-if="false"` 保留占位（如 `FadeIn`、`SequentialTypewriter`），保证产物文件仍存在。

## 正确做法（本仓库标准）

1. **Inline style** 驱动 `opacity` + `translate3d(...px...)` + `transition*`。
2. **`ready` → `play/visible` 两段式**：先 `transition: none` 画隐藏态，再开 transition，再改目标态。
3. 滚动类动效复用 **`useInViewOnce.js`**；使用它的页面必须声明 **`onPageScroll`**。
4. Hero 遮罩揭开：外层 `overflow: hidden` + 固定行高，内层内联上滑；timing 对齐 H5（约 0.6s 起、stagger 0.16s）。
5. 查询失败/超时要有**兜底显示**，禁止永久 `opacity: 0`。
